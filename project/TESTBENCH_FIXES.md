# Scratchpad Testbench Issues & Fixes

## Issues Found in Original `scratchpad_image_tb.sv`

### Critical Issues:

1. **Missing Test Vector Input** ❌
   - The testbench never read the `vectors.sv` file generated by `image_hex_converter.py`
   - Only hardcoded 4-8 pixel patterns instead of full 500×500 image
   - The `read_test_vectors()` function existed but was never called

2. **Insufficient Pixel Count** ❌
   - First test only streamed 4 pixels (lines 128-133)
   - Second test only streamed 8 pixels (lines 147-158)
   - Hardware needs **250,000 pixels** (500×500) to fill the scratchpad
   - This explains why output_rgb.csv showed mostly zeros!

3. **Wrong Pixel Format** ❌
   - Original parsing expected "pixel_index,red,green,blue" format
   - But it parsed `col` incorrectly without mapping pixel_index → row/col
   - Didn't handle the actual hex format from `image_hex_converter.py`

4. **No Synchronization** ❌
   - Didn't wait for all pixels to propagate through the pipeline
   - Captured output immediately after last pixel instead of waiting

### Minor Issues:

5. **No Progress Feedback** ⚠️
   - 250,000 pixel stream with no indication of progress
   - Hard to debug if something hangs

6. **Single Test** ⚠️
   - Only tested the vectors.sv path
   - No fallback validation with simpler patterns

## Potential Hardware Issue ⚠️

Looking at `scratchpad.sv` (lines 60-73):
```systemverilog
always_ff @(posedge clk or negedge rst)
begin
    if(~rst) /* reset */
    else
    begin
        for (int i = 0; i < ROWS; i++)
            for (int j = 0; j < COLS; j++)
            begin
                red_pad[i][j] <= red;    // Broadcasts input to ALL positions!
                green_pad[i][j] <= green;
                blue_pad[i][j] <= blue;
            end
    end
end
```

**This broadcasts the input to ALL 250,000 positions every clock cycle!**

Current behavior:
- Clock 0: All cells get pixel 0
- Clock 1: All cells get pixel 1  
- Clock 249,999: All cells get pixel 249,999
- **Result**: Output matrix is entirely the LAST pixel value

**Expected behavior** (for a proper image scratchpad):
- Should have an address counter or row/col selectors
- Should store each pixel at a unique location
- Should preserve previous values

**Recommendation**: Review the hardware design to verify this is intentional, or implement proper address decoding.
